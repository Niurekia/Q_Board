추천 중복 금지 기능
    vouch = stacked likecount (따봉 중복 X), 1계정 1추천 가능, 그래야만 신뢰도 혹은 보증으로 기능
    사실상 추천만 있어도 되니 비추버튼 굳이?
    추천여부 전용 db를 생성 ->FK로 user db, reply db에서 username, rno 불러온다
    ->그러면 칼럼으로 username, rno, thumb_up 이 구성되고
    추천을 하면 thumb_up => true(혹은 다른거 뭐든)로 바뀌고 더 이상 추천버튼 기능X(혹은 추천 취소)
    (뭔가 고려 못한게 있는거 같은데 모르겠다)
    고려 못한거 1 => thumb_up db에 nothing이라 불러올 값이 없기 때문에 적용이 안되는 중
    그렇다면 최소한 추천 버튼을 누를때 따봉 db에 기본값이 생성되어야
    만약 이미 기본값이 있다면? 무엇을 기준으로 새로 생성이 안되고 기존에 것을 쓸까?
        1.rno==n 인 댓글에 추천 버튼누름
        2.db(=thumb_up)에서 (username==현재 계정명)&&(rno==n) 인 행이 있는지 검사
        3.없다.. => username==현재 계정명, rno==n 인 새 행을 생성하고 thumb_up칼럼 true
            reply db에서 rno==n 인 행의 likecount 값에 +1 해주기
        4.있다! => 해당하는 기존 행의 thumb_up칼럼 true 여부 확인하고
            true이면  reply db에서 rno==n 인 행의 likecount 값에 -1, thumb_up칼럼 false(=not true)
            true아니면 reply db에서.... likecount 값 +1 하고 thumb_up 칼럼 true
    고려 못한거 2 => findById로 특정 행 찾는 방식을 못씀. rno와 username 은 fk이고 같은 값이 여러군데 있을 수 있어서 pk가 아니다
                   어떤 방식으로 username==현재 계정명, rno==n 인 행을 찾을 것인가.
    고려 못한거 3 => 현재 로그인한 계정의 username도 불러와야 행찾을 때 사용가능
